# Experiment 2 — Planar Board vs Marker Cube (쉽게 읽는 설명서)

> **한 줄 요약:** 평면 ChArUco board 만 쓰는 방식과, 바닥 고정 board + 이동하는 marker
> cube 를 쓰는 방식을 **FK(로봇 기구학) 없이 순수 카메라 관측만으로** 비교했다.
> cube 를 추가하면 항상 4대가 같은 타깃을 봐서 → **카메라 위치 오차 −43%, 물체 예측 −34%.**

이 문서는 [fig_exp2_board_vs_cube.png](figures/fig_exp2_board_vs_cube.png) 를 읽는 데 필요한
것을 담는다: **① 두 촬영 시나리오 ② FK 없이 어떻게 캘리브했나 ③ 4개 평가지표 ④ 결과.**

---

## 0. 왜 이 실험을 하나

로봇 셋업에서 카메라들을 캘리브할 때 타깃으로 무엇을 쓰느냐가 관측성을 좌우한다.

- **평면 board (ChArUco)**: 한 면뿐이라 비스듬히 보면 마커가 안 잡힌다 → 소수 카메라만 봄.
- **marker cube**: 6면 모두 마커라 어느 각도에서든 몇 면은 보인다 → 여러 카메라가 동시 관측.

이 관측성 차이가 **실제 캘리브 정확도로 이어지는가**를 시뮬로(정답 GT 를 알고) 검증한다.

---

## 1. 두 촬영 시나리오 (실제 운영 반영)

### A) Board only
- 사람이 ChArUco board 를 **손에 들고 여러 위치·자세로** 옮긴다.
- **최소 2대 카메라가 같은 board 를 동시에 보는** 자세에서만 촬영(사람이 눈으로 확인).
- 이렇게 **총 30장** 촬영.
- → 매 촬영 2~3대만 연결. 카메라 간 관계가 "2대씩" 사슬처럼 **약하다.**

### B) Board + Cube
- **board 는 바닥에 고정** → 4대가 작업공간 바닥을 조금씩 다 비추므로 **항상 4대가 board 를 본다.**
- **cube 는 여러 위치로 이동** → 6면 모두 마커라 어느 각도든 **항상 4대가 cube 도 본다.**
- → 매 촬영 4대가 같은 타깃을 봐서 카메라 간 연결이 **강하고**, cube 다면성으로 시야각이 넓다.

> 핵심 차이: **"한 번에 몇 대가 같은 타깃을 보느냐."**
> A 는 2~3대(약한 연결), B 는 항상 4대(강한 연결).

### 가시성은 기하로 자연 발생 (강제 아님)
"board 는 2대, cube 는 4대"를 손으로 정한 게 아니다. **입사각(카메라 시선과 면 법선의
각도)이 임계(60°)를 넘으면 마커 검출 실패**로 판정한다. 같은 규칙을 board·cube 에 똑같이
적용하니, 평면은 결과적으로 소수만 보고 큐브는 다면성으로 다수가 본다 — **물리적 귀결**이다.

---

## 2. FK 없이 어떻게 캘리브했나

이 실험은 **FK(로봇이 아는 위치)를 전혀 쓰지 않는다.** 오직 카메라가 본 것만으로 캘리브한다.

### 원리: 카메라 합의 (cross-camera consensus)
"여러 카메라가 같은 타깃을 본다"는 사실만으로 카메라들의 상대 위치를 푼다:
```
반복:
  1) 타깃 위치 = 그 타깃을 본 카메라들이 각자 본 것을 base 로 올려 평균 (합의)
  2) 각 카메라 = 그 타깃들로 역산
```
- 한 타깃을 **여러 대가 볼수록**(cube=4대) 합의가 정확해지고 카메라도 정확해진다.
- **2대씩만** 보면(board) 합의가 약해 노이즈가 남는다 → 관측성이 정확도로 직결.

### gauge (절대 기준) 처리
FK 가 없으면 "전체를 어디에 고정하느냐"(gauge)가 정해지지 않는다(회전·평행이동 자유도).
그래서 **카메라 0번을 GT 로 고정**하고 나머지를 상대적으로 정렬한 뒤 GT 와 비교한다.
→ board·cube 를 **같은 gauge** 로 공정하게 비교.

## 3. 평가한 4가지 지표 (figure 4개 패널)

앞 2개(회색) = **관측성**, 뒤 2개(초록) = **실제 캘리브 결과**.

### ① 동시 관측 (cameras / shot) — 관측성 [높을수록 좋음]
촬영 한 번에 몇 대의 카메라가 타깃을 동시에 보나. 카메라 간 연결의 강도.

### ② 시야각 커버리지 (view-angle spread) — 관측성 [높을수록 좋음]
관측 카메라들의 시선이 이루는 각도 분포. 넓을수록 캘리브 조건이 좋다(다양한 방향).

### ③ 카메라 위치 오차 (Camera pose error, vs GT) — 캘리브 결과 [낮을수록 좋음]
순수 카메라 관측으로 추정한 카메라 위치가 **진짜 위치(GT)와 몇 mm 차이나나.**
= "계산된 카메라의 위치가 맞는가". 관측성이 좋으면 이 값이 작아야 한다.

### ④ 타깃(물체) 예측 오차 (Target prediction, vs GT) — 캘리브 결과 [낮을수록 좋음]
캘리브된 카메라들로 타깃(물체) 위치를 base 에서 예측한 오차.
= "특정 물체를 잘 감지했는가". 실전 파지 성능과 직결.

---

## 4. 결과 (30 seed, FK 미사용, 입사각 60°, 30장/방식)

| 지표 | Board only | Board + Cube | 개선 |
|---|---|---|---|
| ① 동시 관측 (대) | 2.63 | **4.00** | |
| ② 시야각 커버리지 (°) | 71.8 | **80.0** | |
| **③ 카메라 위치 오차 (mm)** | 8.31 | **4.73** | **−43%** |
| **④ 타깃 예측 오차 (mm)** | 10.36 | **6.82** | **−34%** |

(캘리브 성공 카메라: 두 방식 다 4대 — 30장이면 board only 도 4대 다 연결됨.)

### 두 가지 메시지
1. **관측성 → 캘리브 정확도로 직결**: board only 는 매 촬영 2.6대만 연결되어(약한 연결)
   카메라 위치 오차가 크지만(8.3mm), board+cube 는 항상 4대가 봐서(강한 연결) 절반으로
   줄어든다(4.7mm, −43%).
2. **물체 감지도 개선**: 그 결과 타깃(물체) 예측도 10.4 → 6.8mm (−34%).
   → cube 를 추가하면 카메라도 물체도 더 정확하게 맞춘다.

### 정직하게 처리한 점
- **FK 를 전혀 쓰지 않았다** — 순수 카메라 관측 합의만으로 캘리브. gauge 는 카메라0 GT 고정.
- 두 방식에 **같은 입사각 규칙·같은 노이즈·같은 gauge** 적용 → 공정 비교.
- 시뮬이라 **진짜 카메라·타깃 위치(GT)를 알기 때문에** ③④ 오차를 직접 잴 수 있다
  (실데이터로는 진짜 카메라 위치를 몰라 불가능).

---

## 5. 왜 board 는 불리하고 cube 는 유리한가 (직관)

- **평면 board**: 한 면뿐 → 비스듬히 보면 마커가 찌그러져 검출 실패 → 정면 근처 1~2대만 봄.
  손에 들고 옮겨도 매번 소수만 봐서 카메라 간 연결이 약하다.
- **marker cube**: 6면 → 어느 각도에서든 카메라를 향한 면이 있다 → 4대가 각자 다른 면을
  동시에 본다. 게다가 바닥 고정 board 까지 항상 보이니 연결이 이중으로 강하다.

> 핵심: **"동시에 같은 타깃을 보는 카메라가 많을수록 상대 위치를 정확히 풀 수 있다."**
> cube 의 다면성이 이 동시 관측 수를 늘려 캘리브 정확도를 높인다.
