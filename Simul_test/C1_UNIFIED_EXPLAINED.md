# C1 통합 캘리브레이션 실험 — 쉽게 읽는 설명서

> **한 줄 요약:** eye-in-hand(그리퍼 카메라)와 eye-to-hand(고정 카메라)를 **하나의 문제로
> 묶어 동시에 최적화(Joint)** 하면, 따로 캘리브해서 붙이는 것보다 훨씬 정확하다.
> (핸드아이 오차 23mm → 0.17mm, 최종 큐브 예측 4.65mm → 0.35mm)

이 문서는 [fig_unified_vs_indep.png](figures/fig_unified_vs_indep.png) 를 읽는 데 필요한
세 가지를 쉽게 설명한다: **① 노이즈를 어떻게 줬나 ② 4개 방법이 각각 뭔가 ③ 4개 평가지표가 뭔가.**

---

## 0. 실험 무대 (셋업)

- **고정 카메라 3대** (eye-to-hand): 작업공간을 둘러싸고 큐브를 바라봄. 안 움직임.
- **그리퍼 카메라 1대** (eye-in-hand): 로봇 팔 끝에 붙어 여러 자세로 움직이며 큐브를 봄.
- **ArUco 마커 큐브**: 두 종류 카메라가 **모두 보는 공유 타깃**. 8개 위치(set)로 옮겨가며 관측.
- 시뮬이라 **정답(GT)** 을 다 안다 → "추정값 vs 진짜값" 오차를 mm 로 직접 잴 수 있다.
  (실데이터로는 진짜 카메라 위치를 몰라 불가능.)

우리가 알아내려는 것:
- **bTf** = 고정 카메라가 base 에서 어디 있나 (3대)
- **gTc** = 그리퍼와 카메라의 상대 위치 (핸드아이, 1개)

설정: 큐브 8 set, train 6 / test 2 holdout, 30 seed 평균.

## 1. 노이즈를 어떻게 만들었나

### 핵심: 이미지를 그리지 않았다
IsaacSim 같은 시뮬레이터로 **이미지를 렌더링하고 마커를 검출한 게 아니다.** 캘리브레이션은
순수하게 **좌표변환(SE(3) 행렬) 기하 문제**라, 이미지 없이 **검출된 큐브 pose 에 노이즈를
직접 주입**했다.

### 3단계로 만든다

**1) 완벽한 관측을 기하로 계산**
GT(진짜 카메라·큐브 위치)를 정해두고, 카메라가 봤어야 할 큐브 pose 를 역산:
```
완벽한 관측 = (카메라 위치)⁻¹ × (큐브 위치)      ← 노이즈 0
```

**2) 실제 검출 오차를 흉내 낸 편향을 더한다**
실제 마커 검출 오차의 **지배 성분**은 매번 랜덤하게 튀는 게 아니라, **큐브 위치에 따라
일정하게 틀리는 systematic(체계적) 편향**이다 (렌즈 왜곡·intrinsic 잔차·작업공간 휨).
그래서 큐브 위치 (x,y)에 **선형 비례하는 편향**을 준다:
```
편향 = G × [큐브의 x, y 위치]        (G = 랜덤하게 정한 3×2 행렬)
```
의미: "큐브가 오른쪽에 있으면 **항상** 오른쪽으로 몇 mm 틀린다" 같은 위치 의존 계통 오차.

**3) 편향을 두 성분으로 나눈다**
```
편향 = G_공통 × [x,y]  +  G_센서별 × [x,y]
```
- **G_공통**: 모든 카메라가 **같은 방향**으로 틀림 (좌표 규약·작업공간 왜곡)
  → 위치의 함수라서 **학습 가능**. 뒤에 나오는 `+fk` 가 이걸 배워서 뺀다.
- **G_센서별**: 카메라마다 **다르게** 틀림 (렌즈·시야각 차)
  → 카메라 간 불일치를 만들어, 실데이터처럼 방법 간 차이가 나게 함.

### 왜 이게 정당한가
캘리브레이션 수식은 **오직 SE(3) 변환의 일관성**만 검증한다. 픽셀·조명·텍스처는 결국
"검출된 큐브 pose + 그 오차"로 요약되고, 우리는 그 요약값을 직접 모델링한다. 이미지를
그리는 건 이 검증에 불필요한 중간 단계다. 그리고 시뮬은 **진짜 카메라 위치를 알기 때문에**
추정 오차를 직접 잴 수 있다(실데이터 불가).

### 실제로 준 노이즈 크기
설정값(`noise_mm=6`)이 관측에 실제로 주는 편향: **평균 ~1.7mm, 최대 ~6mm.**
→ 실제 ArUco 큐브 검출 오차의 **현실적 범위**(보통 2~5mm). 성격도 실제와 같은
systematic(위치의존 계통 오차).

## 2. 실험한 4개 방법

핵심은 **두 축**의 조합이다:
- **통합 여부** (가로): Independent(따로) vs Joint(하나로 묶어 동시 최적화).
- **잔차보정(+fk)** (세로): 캘리브 후 남은 계통오차를 선형 회귀로 학습해 뺄지 여부.

### 통합 여부 — 독립 vs 통합

#### ① Indep (독립) — 따로 풀고 마지막에 붙이기
```
고정 카메라 관측 ─→ [고정 캘리브]  → bTf   ┐
                                          ├→ [공유 큐브로 rigid 정합]
그리퍼 관측 ──────→ [핸드아이]     → gTc   ┘
```
- 고정과 그리퍼가 **서로를 전혀 안 봄.** 각자 자기 관측만으로 캘리브.
- 다 푼 **다음에** 공유 큐브 위치를 기준으로 두 결과를 base 에서 rigid 하게 맞춤(조합 단계).
- **한계**: 각자의 오차(특히 그리퍼 gTc 23mm)가 그대로 남은 채 붙임. 정합이 오차를 못 고침.
- 실제로 흔히 쓰는 방식 → **baseline.**

#### ② Joint (통합) — 처음부터 하나의 문제로 동시 최적화
```
모든 관측(고정+그리퍼) ─→ [하나의 비선형 최소제곱 = 번들 조정]
                          모든 관측이 하나의 일관된 큐브를 가리키도록
                          bTf·gTc·큐브 위치를 전부 동시에 수렴
```
- **애초에 나누지 않는다.** "모든 카메라가 본 큐브가 base 에서 하나의 위치여야 한다"는
  **하나의 목적함수**로 bTf·gTc·큐브를 **동시에** 최적화.
- 정확한 고정 카메라가 최적화 내내 부정확한 그리퍼 gTc 를 끌어올림 → gTc 23→0.17mm.
- **기여점 문서의 "하나의 문제로 구성하여 동시에 최적화"에 정확히 대응.** → **제안 방식.**

> 비유 — 여러 사람이 같은 건물을 각자 찍음:
> Indep=각자 위치 계산 후 겹치기 / **Joint=처음부터 같이 그리기(모든 점을 동시에 맞춤).**

### 잔차보정 (+fk) — 캘리브 후 남은 계통오차 학습
캘리브(기하)를 완벽히 해도, 관측의 systematic 편향은 강체변환으로 설명이 안 돼 남는다.
`+fk` 는 train 큐브의 정답(FK)을 보고 **잔차 = f(큐브 위치)** 를 **선형 회귀(Ridge)** 로
학습해, test 예측에서 그만큼 빼준다.
- **캘리브의 회귀**(비선형, 물리 변환 추정)와 **다른 것**이다: `+fk`는 물리로 설명 안 되는
  잔차를 경험적으로 메우는 것. → 그래서 완벽한 Joint 위에 얹어도 추가 개선(1.72→0.35mm).
- **주의**: `+fk` 는 최종 큐브 예측(다운스트림)에만 작용. bTf·gTc·정합 지표는 안 바뀐다.

### 4개 = 2 통합여부 × {보정 없음, +fk}

| 이름 | 통합 | +fk(잔차보정) |
|---|---|---|
| Indep | 독립(baseline) | ✗ |
| Indep+fk | 독립 | ✓ |
| Joint | 통합(제안) | ✗ |
| **Joint+fk** | 통합 | ✓ (최고 성능) |

## 3. 평가한 4가지 지표 (figure 의 4개 패널)

모두 **"추정값 vs 진짜값(GT)"의 차이(mm)** 다. 그중 **④ 하나만이 실전 성능**이고,
①②③ 은 "왜 그런지"를 설명하는 **진단 계기판**이다.

### ① 고정 카메라 bTf 오차 (eye-to-hand) — 보조/진단
추정한 고정 카메라 위치가 진짜와 몇 mm 차이나나.
- 왜 항상 작나(0.75mm): 고정 카메라는 큐브를 **직접** 봐서 위치가 한 방에 역산됨 + 여러 set
  평균으로 노이즈가 줄어듦.
- 이 패널의 쓸모: 통합(Joint)해도 고정 카메라 정확도가 유지됨(0.75≈0.77mm)을 확인 →
  "통합이 eye-to-hand 를 희생하지 않는다"는 증거.

### ② 핸드아이 gTc 오차 (eye-in-hand) — 핵심 근거 ⭐
추정한 그리퍼–카메라 상대 위치가 진짜와 몇 mm 차이나나.
- 왜 따로 풀면 크나(23mm): 그리퍼는 움직여서 카메라 위치를 직접 못 봄 → 여러 자세의
  **차이(AX=XB)** 로 간접 추정 → 회전-병진이 얽혀 **병진이 구조적으로 부정확**.
  (무노이즈에서도 15~20mm 틀림 = 노이즈가 아니라 문제 구조 탓.)
- **통합의 우수성이 가장 극적으로 드러나는 지표**: Indep 23.10 → **Joint 0.17mm** (−99%).

### ③ 공유 base 정합 일관성 — 핵심 근거 ⭐
고정 카메라로 본 큐브 위치 vs 그리퍼로 본 큐브 위치가 base 에서 얼마나 **일치**하나.
- 통합의 대표 주장("두 카메라가 같은 base 를 공유")을 직접 측정.
- Indep 17.93 → Joint **1.20mm**: 통합할수록 두 서브시스템이 같은 큐브를 같은 데라고 봄.

### ④ 다운스트림 큐브 예측 — 진짜 결과 ⭐⭐
train 으로 캘리브한 뒤 **새 위치(test)의 큐브를 base 에서 얼마나 정확히 맞추나.**
- **이것이 실전 성능이다.** 로봇이 물체를 파지하려면 이 값이 낮아야 한다.
- ②(gTc)가 좋아지면 ④도 좋아진다 → ①②③ 은 ④를 설명하는 원인.

## 4. 결과 한눈에 (systematic 노이즈, 30 seed)

| 방법 | ① 고정 bTf | ② 핸드아이 gTc | ③ 공유정합 | ④ 다운스트림 |
|---|---|---|---|---|
| Indep (baseline) | 0.75 | 23.10 | 17.93 | 4.65 |
| Indep+fk | 0.75 | 23.10 | 17.93 | 4.73 |
| Joint (제안) | 0.77 | **0.17** | **1.20** | 1.72 |
| **Joint+fk** | 0.77 | **0.17** | **1.20** | **0.35** |

(단위 mm, 낮을수록 좋음.)

### 세 가지 메시지
1. **통합(Joint)이 핸드아이를 압도적으로 개선**: gTc 가 Indep 23.10 → Joint **0.17mm**
   (−99%). 동시 최적화가 부정확한 eye-in-hand 를 정확한 eye-to-hand 로 잡아준다.
2. **공유 base 일관성도 통합이 압도**: 17.93 → **1.20mm** (−93%). 두 카메라가 같은 큐브를
   base 에서 같은 데라고 본다 = C1 기여점("공유 base 등록")의 직접 증거.
3. **캘리브를 완벽히 해도 +fk 가 추가로 잡는다**: Joint 1.72 → Joint+fk **0.35mm**.
   물리 최적화(비선형 최소제곱)와 잔차 학습(선형 회귀)은 **다른 오차**를 잡는다
   (→ "선형 회귀면 캘리브랑 같은 것 아니냐" 질문에 대한 반박: 대상이 다르다).
4. **고정 카메라는 손해 없음**: bTf 는 Indep 0.75 ≈ Joint 0.77mm. 통합해도 eye-to-hand
   정확도를 잃지 않으면서 eye-in-hand 를 크게 개선.

## 5. 용어 메모 (헷갈리기 쉬운 것)

- **Joint 최적화** = 비선형 최소제곱 / **번들 조정(bundle adjustment)**. "학습"이 아니라 "추정".
- **+fk (잔차보정)** = **선형 회귀(Ridge regression)**. 데이터에서 잔차 패턴을 "학습".
- 둘 다 최소제곱이지만: 캘리브는 **물리(기하) 파라미터** 추정, +fk 는 **경험적 잔차** 보정.
- "선형 최적화(LP)"와 혼동 금지 — 우리가 쓴 건 최소제곱(회귀), LP 아님.
